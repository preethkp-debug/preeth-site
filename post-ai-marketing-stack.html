The AI marketing stack I'm actually running in 2026.

AI · Growth · 14 min read · May 2026

After eighteen months of testing, here's the prompt library, agent setup and review system I deploy with every client now — and the three things I stopped doing.

prompt ·
agent ·
review

Eighteen months ago I stopped writing about AI in marketing and started actually deploying it. The stack I run with clients today looks almost nothing like the one I started with. This essay is what survived contact with reality.

The premise

Most "AI marketing" advice fails on one of two ends. On one end, it promises that an LLM will replace your strategy team. It won't, and you don't actually want that. On the other, it tells you AI is a slightly better thesaurus. It's much more than that — but only if you build a system around it.

The honest middle is this: a well-designed AI workflow is a 10× multiplier on a workflow you've already designed well, and a 10× amplifier on a workflow you haven't. So the work isn't "use ChatGPT more." It's design the workflow.

The three layers I run for every client

Layer 1 — Prompt libraries

A prompt library is not a folder of prompts. It's a versioned, branded, reviewed set of prompts that produce on-brand output without a senior person rewriting them every time. For each client I build:

  • A brand voice prompt (the constant — appended to everything)
  • 15–25 task prompts (briefs, headlines, ad copy, blog drafts, internal-link suggestions, FAQ generation, transcript cleanup)
  • A small evaluation harness — five sample inputs and a rubric — so we can spot drift when models update

Layer 2 — Agent pipelines

Where prompts are atomic, agents are compositions. The most common pipelines I deploy are:

  • Cluster → Brief → Draft → QA → Publish for editorial
  • Concept → Variants → Score → Library for paid creative
  • Transcript → Summary → Pull-quotes → LinkedIn → Newsletter for repurposing

I keep these in n8n or Make for ops teams, and Notion + Zapier for marketing teams that don't want to think about queues. Both work. The platform matters less than the pipeline.

Layer 3 — Human review loops

Every AI workflow I ship has a human in it. Not as a checkbox, as a gate. The review step is where the system either earns trust or doesn't. Two things make reviews work:

  • Time-boxed. If a piece of content takes more than 8 minutes to review, the prompt is wrong.
  • Loop-closed. Every reviewer note feeds back into the prompt library. Otherwise you're paying for the same correction every week.
"The review step is where the system either earns trust or doesn't."

Three things I stopped doing

1. Long, monolithic prompts

The "mega prompt" was a phase. It's brittle, it hallucinates, it can't be debugged. I now break every workflow into the smallest reasonable steps, each with one job.

2. Output without QA

For about three months I shipped AI content with the kind of cursory check that would never have passed for a human writer. Two clients got bitten by factual errors. I now treat every AI output as a "junior writer with no domain knowledge" — which is the correct mental model.

3. Tool-of-the-month

The LinkedIn AI grift cycle is real. New tools every Tuesday. I now stay on a small stack — Claude (daily driver), one image model, n8n for orchestration, Notion for storage — and only adopt new tools when an existing pipeline visibly breaks.

What's actually in the stack

For full transparency, here's what's running in production for most of my clients today, as of May 2026:

  • Reasoning: Claude Opus 4.6 for strategic work; Claude Sonnet 4.6 for high-volume drafting
  • Search-grounded: Perplexity API for fact-checked outputs; ChatGPT for SERP-style answers
  • Images: Midjourney for hero visuals; Adobe Firefly for ad-safe variants
  • Orchestration: n8n self-hosted; Zapier for low-volume glue
  • Storage & review: Notion (briefs, prompt library, retros); Linear (engineering tickets); Slack for human-in-the-loop
  • Measurement: PostHog for behaviour, Looker Studio for marketing dashboards, BigQuery for cohorts

Where this is going

The next twelve months are about agents that act, not just generate. I'm already running three pilots — autonomous bid optimisation, autonomous content QA, autonomous LinkedIn outreach. None of them are ready for prime-time. But the direction is clear: less prompting, more guardrails. Less generation, more orchestration.
